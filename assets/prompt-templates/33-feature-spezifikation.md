# Prompt-Template 33: Feature-Spezifikation

## Zweck
Klare Anforderungen für Entwickler und Stakeholder.

## Prompt
```
ROLLE: Als Product Manager...

KONTEXT:
Feature: [Name]
Problem: [Was lösen wir?]
Nutzer: [Wer profitiert?]
Business Impact: [Warum jetzt?]

AUFGABE: Schreibe eine Feature-Spezifikation.

FORMAT:
1. Overview: 2-Satz-Zusammenfassung
2. Problem Statement: User Story + Pain Point
3. Lösung: Beschreibung der Funktionalität
4. Acceptance Criteria: 5-7 messbare Kriterien
5. Design: Mockup-Beschreibung oder Link
6. Tech Notes: Einschränkungen oder Dependencies
7. Success Metrics: Wie messen wir Erfolg?
8. Rollout Plan: Phasen oder A/B-Test
```

## Variationen
- Für Agile: User Stories und Acceptance Criteria
- Für Enterprise: Compliance und Security-Anforderungen
- Für MVP: Scope reduzieren auf Minimum
