# Prompt-Template 37: User Story

## Zweck
Nutzer-zentrierte Anforderungen für agile Teams.

## Prompt
```
ROLLE: Als Product Owner...

KONTEXT:
Nutzer: [Wer ist der Akteur?]
Bedürfnis: [Was will er erreichen?]
Nutzen: [Warum ist das wichtig?]

AUFGABE: Formuliere eine User Story mit Acceptance Criteria.

FORMAT:
- Story: Als [Rolle] möchte ich [Ziel], damit [Nutzen]
- Beschreibung: Kontext und Hintergrund
- Acceptance Criteria: 3-5 messbare Kriterien (Given/When/Then)
- Definition of Done: Was muss erfüllt sein?
- Schätzung: Story Points oder Zeit
- Abhängigkeiten: Blocker oder Voraussetzungen
```

## Variationen
- Für Epics: Mehrere Stories zusammenfassen
- Für Bugs: Problem und Lösung formulieren
- Für Tech Debt: Refactoring-Begründung
