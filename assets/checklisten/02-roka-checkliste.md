# Checkliste: ROKA-Rahmen anwenden

## Schritt-für-Schritt durch den ROKA-Rahmen

### R - ROLLE

**Checkliste:**
- [ ] Spezifische Rolle definiert (nicht "Experte", sondern "SEO-Experte für E-Commerce")
- [ ] Erfahrungslevel angegeben ("mit 5 Jahren Erfahrung")
- [ ] Zielgruppe der Antwort genannt ("für Anfänger", "für Fachleute")

**Formel:**
```
Als [spezifische Rolle] mit [Erfahrung/Kontext] für [Zielgruppe/Audienz]...
```

**Beispiele:**
- ✅ "Als erfahrener UX-Designer mit Fokus auf Conversion-Optimierung..."
- ✅ "Als kritischer Business-Journalist für ein gebildetes Laienpublikum..."
- ❌ "Als Experte..."
- ❌ "Als KI..."

---

### O - ORGANISATION (Kontext)

**Checkliste:**
- [ ] Wer ist der Empfänger?
- [ ] Was ist das Ziel?
- [ ] Was ist der Hintergrund?
- [ ] Gibt es Einschränkungen?
- [ ] Was wurde bereits getan?
- [ ] Welcher Ton ist angemessen?

**Struktur:**
```
- Wir sind [Unternehmen/Projekt]
- Unsere Zielgruppe ist [Beschreibung]
- Wir wollen [Ziel]
- Unsere Constraints sind [Einschränkungen]
- Bisher haben wir [Status Quo]
```

**Beispiel:**
```
- Wir sind ein SaaS-Startup mit 20 Mitarbeitern
- Unser Produkt hilft bei der Zeiterfassung
- Wir wollen in den Enterprise-Markt
- Unsere aktuelle Website spricht kleine Teams an
- Budget für Redesign: 10k€
```

---

### K - KONKRETE AUFGABE

**Checkliste:**
- [ ] Eindeutiges Verb am Anfang
- [ ] Klares Objekt (was genau?)
- [ ] Parameter definiert (wie viel, wie lange)
- [ ] Ziel angegeben (was soll erreicht werden?)
- [ ] Nur eine Hauptaufgabe

**Formel:**
```
[Verb] + [Objekt] + [Parameter] + [Ziel]
```

**Gute Verben:**
- Erstelle
- Analysiere
- Optimiere
- Strukturiere
- Übersetze
- Fasse zusammen
- Vergleiche
- Empfehle

**Beispiele:**
- ✅ "Erstelle eine Content-Strategie für 3 Monate, die zu 20% mehr Leads führt."
- ✅ "Analysiere diese E-Mail auf Verkaufspotenzial und identifiziere 3 Verbesserungspunkte."
- ❌ "Mach was mit Marketing."
- ❌ "Hilf mir bei diesem Problem."

---

### A - AUSGABEFORMAT

**Checkliste:**
- [ ] Struktur definiert (Liste, Tabelle, Fließtext)
- [ ] Länge angegeben (Wörter, Sätze, Zeichen)
- [ ] Ton beschrieben (formell, locker, professionell)
- [ ] Stil festgelegt (kurz, ausführlich, bullet-points)
- [ ] Besondere Anforderungen (keine Füllwörter, Fachbegriffe erklären)

**Struktur:**
```
Format:
- [Struktur: Liste/Tabelle/Text]
- [Länge: X Wörter/Sätze]
- [Ton: beschreibend]
- [Stil: beschreibend]
- [Besonderheiten]
```

**Beispiel:**
```
Format:
- Tabelle mit 3 Spalten: Problem, Lösung, Zeitaufwand
- Maximal 10 Zeilen
- Ton: pragmatisch, ohne Füllwörter
- Fachbegriffe in Klammern erklären
```

## Komplettes ROKA-Beispiel

```
ROLLE:
Als erfahrener SEO-Experte mit Fokus auf 
E-Commerce für einen Online-Shop-Betreiber, 
der gerade erst anfängt.

ORGANISATION (Kontext):
- Wir verkaufen nachhaltige Haushaltsprodukte
- Der Shop läuft seit 6 Monaten
- Aktueller Traffic: 500 Besucher/Monat
- Ziel: 5000 Besucher/Monat in 12 Monaten
- Budget für SEO-Tools: 100€/Monat
- Kein Team, alles selbst gemacht

KONKRETE AUFGABE:
Erstelle einen 90-Tage-SEO-Plan mit konkreten 
Aktionen, die ohne teure Tools umsetzbar sind. 
Jede Aktion soll einen geschätzten Impact haben 
(low/medium/high).

AUSGABEFORMAT:
- Wochenweise Aufschlüsselung (Woche 1-12)
- Pro Woche: 3 konkrete Aktionen
- Jede Aktion: Was? Wie? Zeitaufwand? Impact?
- Zusammenfassung am Ende: Top 3 Prioritäten
- Ton: motivierend, aber realistisch
```

## Häufige ROKA-Fehler

| Fehler | Korrektur |
|--------|-----------|
| Rolle zu allgemein | Spezifizieren: Branche, Erfahrung |
| Zu viel Kontext | Auf 3-5 relevante Punkte reduzieren |
| Aufgabe unklar | Verb + Objekt + Parameter |
| Format fehlt | Struktur, Länge, Ton definieren |
| Mehrere Aufgaben | Aufteilen in mehrere Prompts |

---

*Nutze diese Checkliste für jeden wichtigen Prompt. Es lohnt sich.*
