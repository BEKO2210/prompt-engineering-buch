# Kapitel 7: Praxisbeispiele

## 20+ Prompts für den Alltag

Hier sind Prompts, die ich selbst nutze. Kopiere sie, passe sie an, mach sie zu deinen.

## Content & Marketing

### 1. Blogartikel schreiben
```
ROLLE: Als erfahrener Content-Marketer für 
[Branche]...

KONTEXT: Wir müssen einen Blogartikel über 
[Thema] erstellen. Zielgruppe: [Beschreibung]. 
Ziel: [Ziel des Artikels].

AUFGABE: Schreibe einen Blogartikel (800-1000 
Wörter) mit folgender Struktur:
1. Einleitung mit Hook
2. 3-4 Hauptabschnitte mit Zwischenüberschriften
3. Praktische Tipps/Actionable Advice
4. Fazit mit Call-to-Action

FORMAT:
- Ton: [professionell/locker/enthusiastisch]
- Zwischenüberschriften: H2
- Bullet Points für Listen
- Fachbegriffe erklären
```

### 2. Social Media Content
```
Erstelle 5 LinkedIn-Posts aus diesem 
Blogartikel: [Artikel einfügen]

Anforderungen pro Post:
- Maximal 150 Wörter
- Ein Hook in den ersten 2 Zeilen
- Ein praktischer Takeaway
- Ein Call-to-Action am Ende
- 3-5 relevante Hashtags

Tonalität: [wählbar]
```

### 3. E-Mail Sequenz
```
ROLLE: Als E-Mail-Marketing-Experte...

KONTEXT: Wir launch ein neues Produkt: 
[Produktbeschreibung]. Zielgruppe: [Zielgruppe].

AUFGABE: Erstelle eine 5-teilige Launch-E-Mail-
Sequenz:
- E-Mail 1: Teaser (1 Woche vor Launch)
- E-Mail 2: Early-Bird-Angebot (3 Tage vorher)
- E-Mail 3: Launch-Day
- E-Mail 4: Social Proof + Dringlichkeit
- E-Mail 5: Last Call

FORMAT:
- Betreffzeile für jede E-Mail
- Preview-Text
- E-Mail-Body (max. 200 Wörter)
- Call-to-Action
```

### 4. Ad Copy
```
Schreibe 5 Google Ads-Anzeigen für 
[Produkt/Dienstleistung].

Anforderungen:
- Headline 1: 30 Zeichen
- Headline 2: 30 Zeichen
- Headline 3: 30 Zeichen
- Beschreibung 1: 90 Zeichen
- Beschreibung 2: 90 Zeichen

Jede Anzeige soll einen anderen Ansatz haben:
1. Nutzen-orientiert
2. Schmerzpunkt-adressierend
3. Preis/Leistung
4. Dringlichkeit
5. Social Proof
```

## Business & Strategie

### 5. Business-Plan-Analyse
```
Analysiere diesen Business-Plan-Abschnitt:
[Text einfügen]

Bewerte nach folgenden Kriterien (1-10):
- Klarheit der Zielgruppe
- Einzigartigkeit des Angebots
- Machbarkeit der Strategie
- Finanzielle Planung

Gib für jede Bewertung:
- Begründung
- Konkreten Verbesserungsvorschlag
```

### 6. Wettbewerbsanalyse
```
Analysiere diese 3 Wettbewerber:
[Wettbewerber 1, 2, 3]

Erstelle eine Vergleichstabelle mit:
- Positionierung
- Preismodell
- Stärken
- Schwächen
- Unique Selling Points

Am Ende: 3 Empfehlungen, wie wir uns 
differenzieren können.
```

### 7. Pitch Deck Review
```
Review dieses Pitch-Deck (Folien als Text):
[Text einfügen]

Bewerte:
- Storyline (logischer Aufbau?)
- Problem-Lösung-Fit
- Marktgröße (realistisch?)
- Business Model (klar?)
- Team (relevante Erfahrung?)
- Ask (klar formuliert?)

Gib 5 konkrete Verbesserungsvorschläge.
```

## Produktivität & Organisation

### 8. Meeting-Zusammenfassung
```
Fasse dieses Meeting-Transkript zusammen:
[Transkript einfügen]

FORMAT:
- Teilnehmer: [Liste]
- Hauptthemen (max. 3)
- Getroffene Entscheidungen
- Offene Punkte mit Verantwortlichen
- Nächste Schritte mit Deadlines

Maximal 200 Wörter.
```

### 9. To-Do-Priorisierung
```
Hier ist meine To-Do-Liste:
[Liste einfügen]

Kontext:
- Heute ist [Datum]
- Diese Woche habe ich [X Stunden] verfügbar
- Meine Top-Priorität ist: [Priorität]

AUFGABE: Priorisiere die Liste nach:
1. Dringlichkeit (wann fällig?)
2. Wichtigkeit (Impact auf Ziele)
3. Zeitaufwand

Ergebnis: Nummerierte Liste mit Begründung 
pro Priorität.
```

### 10. Wochenplanung
```
Erstelle einen Wochenplan basierend auf:

Meine Ziele diese Woche:
[Ziele]

Fixe Termine:
[Termine]

Verfügbare Arbeitszeit:
[Zeitfenster]

Erstelle einen Tag-für-Tag-Plan mit:
- Zeitblöcken für Fokusarbeit
- Pufferzeiten
- Review-Zeit am Freitag
```

## Kommunikation

### 11. Schwierige E-Mail
```
Kontext: [Was ist passiert?]
Empfänger: [Wer? Beziehung?]
Ziel: [Was soll erreicht werden?]

Schreibe eine E-Mail, die:
- Den Sachverhalt klar darstellt
- Empathie zeigt (aber nicht zu viel)
- Eine Lösung anbietet
- Professionell bleibt
- Nicht entschuldigend klingt

Maximal 150 Wörter.
```

### 12. Feedback formulieren
```
Ich muss Feedback geben zu: [Situation]

An wen: [Person, Rolle, Erfahrungslevel]

Das Feedback soll:
- Konstruktiv sein
- Spezifische Beispiele enthalten
- Verbesserungsvorschläge bieten
- Motivierend enden

Vermeide:
- Persönliche Kritik
- Vage Aussagen
- Zu viele Punkte auf einmal
```

### 13. Verhandlungsstrategie
```
Ich verhandle über: [Was?]

Meine Position:
- Ideal: [Ziel]
- Walk-away: [Minimum]
- BATNA: [Alternative]

Gegenüber vermutlich:
- Will: [Vermutung]
- Druck: [Zeit/Geld/Alternativen]

Entwickle eine Verhandlungsstrategie:
1. Erstes Angebot (Anchor)
2. Konzessionen (Reihenfolge)
3. Einwände und Gegenargumente
4. Closing-Strategie
```

## Lernen & Recherche

### 14. Komplexes Thema erklärt
```
Erkläre [komplexes Thema] so, dass ein 
[12-Jähriger/Anfänger/Nicht-Techniker] 
es versteht.

Nutze:
- Ein Alltagsbeispiel
- Eine Analogie
- Maximal 150 Wörter
- Keine Fachbegriffe (oder erklärt)
```

### 15. Buch-Zusammenfassung
```
Fasse das Buch [Titel] zusammen.

FORMAT:
- Kernthese (1 Satz)
- 3 Hauptideen mit Erklärung
- 3 praktische Anwendungen
- Für wen das Buch geeignet ist
- Mein Fazit (lesenswert ja/nein)

Maximal 300 Wörter.
```

### 16. Studie analysieren
```
Analysiere diese Studie:
[Zusammenfassung oder Abstract]

Bewerte:
- Methodik (stark/schwach?)
- Stichprobengröße (ausreichend?)
- Aussagekraft (kausal/korrelativ?)
- Limitationen
- Praktische Relevanz

Für wen ist diese Studie relevant?
```

## Technik & Entwicklung

### 17. Code erklärt
```
Erkläre diesen Code:
[Code einfügen]

Für wen: [Anfänger/Fortgeschrittener]

Erkläre:
- Was macht der Code?
- Wie funktioniert er Schritt für Schritt?
- Was sind potenzielle Probleme?
- Wie könnte man ihn verbessern?
```

### 18. Bug-Analyse
```
Ich habe einen Bug:

Fehlermeldung: [Fehler]
Kontext: [Was sollte passieren?]
Code: [Relevanter Code]

Analysiere:
- Mögliche Ursachen
- Debugging-Schritte
- Lösungsvorschläge
- Prävention für die Zukunft
```

### 19. Code-Review
```
Review diesen Code:
[Code einfügen]

Bewerte:
- Lesbarkeit
- Effizienz
- Best Practices
- Sicherheit
- Testbarkeit

Gib für jeden Punkt:
- Bewertung (gut/mittel/schlecht)
- Begründung
- Verbesserungsvorschlag (falls nötig)
```

## Kreativität

### 20. Brainstorming
```
Brainstorming für: [Thema/Problem]

Kriterien:
- Mindestens 10 Ideen
- Verschiedene Ansätze (wild/pragmatisch/teuer/billig)
- Keine Filterung während des Brainstormings
- Kurze Beschreibung pro Idee

Am Ende: Markiere die 3 vielversprechendsten.
```

### 21. Namensfindung
```
Generiere Namen für [Projekt/Produkt/Unternehmen].

Anforderungen:
- [Kriterium 1, z.B. kurz, einprägsam]
- [Kriterium 2, z.B. Domain verfügbar]
- [Kriterium 3, z.B. international verständlich]

10 Vorschläge mit:
- Name
- Bedeutung/Herkunft
- Warum er passt
- Potenzielle Domain
```

### 22. Storytelling
```
Erstelle eine Geschichte über [Thema].

Elemente:
- Protagonist mit Problem
- Konflikt/Eskalation
- Wendepunkt
- Auflösung
- Moral/Learning

Zielgruppe: [Zielgruppe]
Länge: [Wörter/Sätze]
Ton: [Stimmung]
```

## Finanzen

### 23. Finanzplan-Review
```
Review diesen Finanzplan:
[Zahlen einfügen]

Analysiere:
- Annahmen (realistisch?)
- Cashflow (Engpässe?)
- Break-even (wann erreicht?)
- Risiken
- Vergleich mit Industry Benchmarks

Gib 3 Empfehlungen zur Verbesserung.
```

### 24. Investitionsentscheidung
```
Ich überlege zu investieren in: [Investment]

Meine Situation:
- Risikotoleranz: [niedrig/mittel/hoch]
- Zeithorizont: [kurz/mittel/lang]
- Budget: [Betrag]

Pro-Argumente: [Liste]
Contra-Argumente: [Liste]

Analysiere objektiv:
- Risiko-Rendite-Profil
- Passend zu meiner Situation?
- Alternativen
- Empfehlung mit Begründung
```

## Zusammenfassung

Diese Prompts sind Startpunkte. Passe sie an deine Situation an. Je spezifischer, desto besser das Ergebnis.

Im nächsten Kapitel schauen wir uns die besten Tools an – und wie du Workflows baust, die skalieren.
